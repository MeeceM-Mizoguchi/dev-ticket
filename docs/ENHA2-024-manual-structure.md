# ENHA2-024 マニュアル表示機能 ― 詳細構成（ページ設計）確定版

業務フロー順・全8部。各章を「ページ（ステップ）」単位まで分解した設計表。
本マニュアルは**静的なドキュメント**（上から読む形式）。各ステップは「説明文＋コーディング再現した画面（画像貼り付けサイズ感）」で構成し、画面上の対象箇所を**赤枠で囲み、それ以外をグレーアウト**して強調する。アニメーション等の動く演出はなし。

- `表示画面` = `screens/` 配下に作るコーディング再現モック
- `強調箇所` = 対象以外グレーアウト＋対象に赤枠で示す位置（該当なしは「—」）
- `権限` = その操作を使えるユーザー。権限設定なく誰でも使えるものは「全ユーザー」。権限が必要なものは **設定場所（ロール設定／アサイン計画）＋権限名** で記載
- ※ Owner専用の機能（組織作成など）は社内共有のためマニュアル対象外

---

## 権限による出し分け仕様（案A・確定）

マニュアルは全ユーザーがアクセスできるが、**各章・各ステップは「権限」列の権限を持つ人にだけ表示**する（持たない人にはその章・ステップ自体が表示されず、TOCからも消える）。判定は次の2系統：

1. **ロール設定系の権限**（メンバー管理／ロール設定／アサイン計画／通知管理／レポート管理／お知らせ更新／レビュースキップ）
   → `useAuth().userPermissions`（ロールの base_permissions 由来）で即判定。持っていれば表示。

2. **アサイン計画系の権限**（チケット作成／スプリント作成／編集・削除／レビュー権限／バックログ・Wiki・議事録・ホワイトボード）
   → プロジェクトごとに `project_member_permissions` に保存される。**案A採用**：ユーザーの全参加プロジェクトの権限をロードしOR集約し、**どれか1つのプロジェクトで「あり（編集/作成等）」なら表示**する。実装は集約ヘルパー（例 `useAggregatedProjectPermissions()`）で対応。DB変更は不要。

- 「全ユーザー」の章・ステップは常に全員に表示。
- 台帳（`manualRegistry`）の各章・各ステップに `requiredPermission`（無指定＝全ユーザー）を持たせ、表示時に上記で照合してフィルタする（サイドバーの表示制御と同方式）。

## 権限ラベルの凡例（設定場所つき）

**ロール設定で付与する権限**（管理画面へのアクセス制御）
| 権限名（ロール設定の表示） | 対応機能 |
|---|---|
| メンバー管理 | メンバー招待・編集（2-1） |
| ロール設定 | ロール設定（2-2） |
| アサイン計画 | アサイン計画（2-3） |
| 通知管理 | Slack連携・通知管理（7-1） |
| レポート管理 | レポート管理（7-3） |
| お知らせ更新 | お知らせ設定（7-2）※Meece組織のみ |
| レビュースキップ | レビュー省略（4-6） |

**アサイン計画（グループ／メンバー）で付与する権限**（プロジェクト内の操作）
| 権限名（アサイン計画の表示） | 対応機能 |
|---|---|
| チケット作成 | チケット作成（3-3, 4-5, 5-1） |
| スプリント作成 | スプリント作成（3-2） |
| 編集・削除 | チケットの編集・ステータス変更全般（4-2〜4-3, 4-7〜4-10） |
| レビュー権限 | レビュー承認・差し戻し（4-6） |
| バックログ（なし/閲覧のみ/編集） | バックログ（5-1） |
| Wiki（なし/閲覧のみ/編集） | Wiki（5-2） |
| 議事録（なし/閲覧のみ/編集） | 議事録（5-3） |
| ホワイトボード（なし/閲覧のみ/編集） | ホワイトボード（5-4） |

**その他**
| ラベル | 意味 |
|---|---|
| 全ユーザー | 権限設定なく誰でも利用可（表示結果は所属・参加範囲でフィルタ）。チケットの閲覧・コメントは常に可能 |
| 管理者・PMロールの人 | プロジェクト作成・クライアント管理など、ロール（管理者/PM）に紐づく操作（トグル権限ではない） |
